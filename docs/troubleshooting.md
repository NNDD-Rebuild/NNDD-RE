# トラブルシューティング

## 動画の再生とダウンロード、必要なツールが違う

再生（ストリーミング、デフォルト動作）とダウンロード（ライブラリへの保存）は別経路で、必要な外部ツールが異なります。

**再生（yt-dlp / FFmpeg 不要）**

```
WatchInfoHandler → 視聴ページ解析
       ↓
WatchSession     → HLSアクセス権取得（最高品質を自動選択）
       ↓
hls.js           → アプリ内に直接ストリーミング再生
```

設定のストリーミング再生モードにより挙動が変わります。いずれのモードも yt-dlp・FFmpeg は不要です。

- `native`（デフォルト）: hls.js が niconico CDN に直接アクセス
- `hls`: 内蔵HTTPプロキシ経由で再生
- `niconico`: ニコニコ公式プレイヤーを埋め込み表示（音声のみ再生は非対応）

**ダウンロード（yt-dlp + FFmpeg 必須）**

```
DownloadManager.enqueue()
       ↓
yt-dlp が bestvideo+bestaudio を取得
       ↓
FFmpeg（yt-dlpが自動検出して呼び出し） → mp4へmux
       ↓
ライブラリへ登録・ファイル命名
```

## Q. 動画がダウンロードできない

設定 → 外部ツール画面から yt-dlp / FFmpeg をダウンロードしてください。バイナリは `userData/bin/` 配下に保存されます。yt-dlpと同じディレクトリに置くことで、yt-dlpがFFmpegを自動検出します。

コメントのみダウンロード（`commentOnly: true`）の場合はこの動画DLステップ自体がスキップされ、yt-dlp/FFmpegは呼び出されません。yt-dlpが見つからない場合はDL項目が失敗扱いになるだけで、アプリ自体や他の機能には影響しません。

## Q. 動画は視聴できるのにダウンロードだけ動かない

視聴（デフォルトのストリーミング再生）は niconico CDN から `hls.js` で直接再生するため yt-dlp / FFmpeg は不要ですが、ダウンロードにはこの2つが必須です。上記「動画がダウンロードできない」の手順で導入してください。

## その他

- `better-sqlite3` はネイティブモジュールのため、環境によっては再ビルドが必要になることがあります（開発者向け）
- カスタムプロトコル `nndd-re-local://` はローカルファイル再生用に安全に許可されています
